# 프로젝트 비전

이 프로젝트의 목표는 원신 자료 검색기를 만드는 것이 아니라, 공식 텍스트 기반 스토리 연구 AI를 만들기 위한 기반을 구축하는 것입니다.

사용자는 최종적으로 ChatGPT처럼 자연어로 질문합니다.

```text
파네스랑 천리의 주관자는 같은 존재일 가능성이 있어?
세계수와 기억 조작 떡밥이 천리와 연결될 수 있어?
페이몬의 정체를 공식 근거 중심으로 다시 봐줘.
```

시스템은 단순히 질문에 들어간 단어만 검색하지 않습니다. 관련 엔티티, 다국어 별칭, 주변 개념, 반복 모티프, 번역 차이, 반례 후보까지 탐색한 뒤 공식 근거와 추측을 분리해서 답변하는 것을 목표로 합니다.

## 핵심 목표

```text
질문
→ 질문 의도 분석
→ 엔티티 추출
→ 다국어 별칭 확장
→ 공식 원문 검색
→ 모티프와 주변 개념 탐색
→ 번역 차이 확인
→ 반례 후보 검색
→ 여러 가설 비교
→ Evidence Pack 생성
→ 답변 생성
→ 연구 메모리 업데이트
```

즉, 일반적인 RAG처럼 `질문 → 관련 청크 검색 → 답변`으로 끝내지 않습니다. 목표는 AI가 원신 공식 DB를 탐험 가능한 세계지도처럼 사용하고, 그 위에서 가설을 만들고 검증하게 하는 것입니다.

## 유기적 연결

이 프로젝트에서 말하는 연결은 단순한 동시 등장보다 넓습니다.

```text
동일성 연결
계승 연결
상징 연결
대립 연결
원인-결과 연결
같은 체계 연결
번역상 연결
서사 구조 연결
시간대 연결
사용자 가설상 연결
```

예를 들어 `파네스`와 `천리`의 관계도 하나로 단정하지 않습니다.

```text
A. 파네스 = 천리 동일 존재설
B. 파네스 체계 → 천리 체계 계승설
C. 같은 질서 계열의 다른 존재설
D. 창조자-관리자 관계설
E. 상징적으로 유사하지만 직접 관계는 약한 관계
```

AI는 가능한 관계 후보를 나누고, 각 후보의 근거와 약점을 같이 제시해야 합니다.

## 출처 계층

답변에서는 공식 근거와 추측을 반드시 분리합니다.

```text
L0: 공식 원문 DB
L1: 공식 텍스트에서 자동 추출한 관계
L2: AI 연구 메모리 / 가설
L3: 위키 또는 정리 자료
L4: 커뮤니티 추측
L5: AI 자체 추론
```

공식 텍스트에 직접 있는 내용과 AI가 모티프를 바탕으로 만든 연결을 같은 톤으로 말하면 안 됩니다.

## 연구 메모리

최종 시스템은 채팅 또는 그룹별 연구 메모리를 가질 수 있습니다.

```text
중심 가설
병렬 가설
임시 관점
약화된 가설
반박된 가설
폐기된 가설
사용자 선호
반례 노트
```

사용자가 새로운 관점을 제시한다고 해서 기존 가설을 즉시 폐기하지 않습니다. `이런 관점도 봐볼까?`는 병렬 가설로, `그건 아닌 것 같고 앞으로 이쪽으로 가자`는 중심 가설 전환 후보로 해석하는 식입니다.

## 최종 정의

이 프로젝트는 다음과 같이 정의할 수 있습니다.

> Project Amber 전체 덤프와 다국어 공식 텍스트를 기반으로, AI가 원신 세계관 자료를 능동적으로 탐색하고, 엔티티·모티프·번역 차이·반복 구조를 연결하며, 여러 스토리 가설을 근거와 반례로 비교하는 ChatGPT형 대화형 원신 스토리 연구 AI.
